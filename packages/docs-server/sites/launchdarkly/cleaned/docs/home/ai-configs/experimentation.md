## Overview
This topic introduces the role of AI Configs in LaunchDarkly Experimentation. LaunchDarklyâ€™s Experimentation feature lets you measure the effect of features on end users by tracking metrics your team cares about. By connecting metrics you create to AI Configs in your LaunchDarkly environment, you can measure changes in customer behavior based on the different AI Config variations your application serves. This helps you make more informed decisions and ensures that the features your development team ships align with your business objectives.
### Monitoring and Experimentation
Each AI Config includes a **Monitoring** tab in the LaunchDarkly user interface (UI). This tab displays performance data if you [track AI metrics in your SDK](/docs/sdk/features/ai-metrics), such as input and output tokens or total call duration to your LLM provider. To learn more, read [Monitor AI Configs](/docs/home/ai-configs/monitor).
In contrast, Experimentation lets you measure how your application changes affect end user behavior, based on signals like page views and clicks. For example, you might use the **Monitoring** tab of an AI Config to identify which variation consumes the fewest output tokens. But to determine which variation results in the most clicks in your chatbot, you need to run an experiment.
To get started with Experimentation in the context of AI Configs, explore the following resources:
 * [Experimentation](/docs/home/experimentation) reference
 * [Metrics](/docs/home/metrics) reference
 * [Experimentation guides](/docs/guides/experimentation), including best practices and background on statistical methods
##### Guarded rollouts and experiments cannot run at the same time
You cannot run a [guarded rollout](/docs/home/releases/guarded-rollouts) and an [experiment](/docs/home/experimentation) on the same flag at the same time. To monitor a variation rollout for regressions, use a guarded rollout. To measure how a variation affects a metric, use an experiment.
