## Overview
This category lists persistent store integrations available to use with big segments.
A big segment is a segment that is either a [synced segment](/docs/home/flags/synced-segments), or a [list-based segment](/docs/home/flags/list-based-segments) with more than 15,000 entries that includes only one targeted [context kind](/docs/home/flags/context-kinds). LaunchDarkly uses different implementations for different types of segments so that all of your segments have good performance.
If you are working with server-side SDKs, both synced segments and larger list-based segments require a persistent store integration. You must configure this integration for each environment where you create these segments.
LaunchDarkly supports the following persistent store integration for use with big segments:
 * [DynamoDB](/docs/integrations/dynamodb)
 * [Redis](/docs/integrations/redis)
