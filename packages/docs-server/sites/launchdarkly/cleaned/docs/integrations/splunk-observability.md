##### Splunk Observability Cloud was formerly SignalFx
Splunk Observability Cloud used to be called SignalFx. Some features of Splunk Observability Cloud, such as API endpoints, still reference SignalFx.
## Overview
This topic explains the LaunchDarkly Splunk Observability Cloud integrations available, including events and triggers.
[Splunk Observability Cloud](https://www.splunk.com/en_us/products/observability.html) is an [application performance management (APM)](https://en.m.wikipedia.org/wiki/Application_performance_management) tool. It allows customers to monitor the operational health of their application and infrastructure.
## Events
##### The Splunk Observability Cloud events integration is available to customers on select plans
The Splunk Observability Cloud events integration is only available to customers on select plans. To learn more, [read about our pricing](https://launchdarkly.com/pricing/). To upgrade your plan, [contact Sales](https://launchdarkly.com/contact-sales/).
The LaunchDarkly Splunk Observability Cloud events integration exposes flag change data to Splunk Observability Cloud, which can detect and alert engineers when their applications are trending in an unstable direction. To learn how to set up this integration, read [Using the Splunk Observability Cloud events integration](/docs/integrations/splunk-observability/events).
## Triggers
##### Flag triggers are available to customers on select plans
Flag triggers are only available to customers on select plans. To learn more, [read about our pricing](https://launchdarkly.com/pricing/). To upgrade your plan, [contact Sales](https://launchdarkly.com/contact-sales/).
